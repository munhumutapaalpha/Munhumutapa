Polls & Voting

Voting System

The Munhumutapa Voting System allows users to create polls on which other blockchain users can vote on with results and real-time stats publicly viewable, tracked and verified. Poll creators can set eligibility requirements for voter participants which gives polls fairness, transaparency, credibility and security.

Any user of the blockchain can create a poll.

Poll Creation

Any user account can create a poll and this poll can be viewed by anyone including Munhumutapa non-users. However, to participate in a poll, a Munhumutapa account is required and also needs to match the eligibility set by the poll creator. Every poll has configurable properties which allow poll creators to customise polls depending on the poll creators preferences, poll scope and target demograph.

Poll Properties

Poll Description

The description of a poll explains in detail its purpose, scope and any other information the creator may want to communicate to an observer or poll participant. The description can also reference external links.

Poll Duration

Each poll has a start height at which it is open for voting and a a finish heightafter which votes are can no longer be cast. These height parameters are required for every poll and are set af poll creation.

Poll Options

A poll creator can set predefined options to which voters and participants can vote for their choice from a pre-defined set of options. Options can be any text from one-word choices to numbers or brief summaries. Also, a poll can also be open for participants to give general answers in their own words.

Poll Eligibility

Poll creators can select participants by setting conditions for which an account should meet to be considered eligible to take part in a poll. These conditions are limited to those that can be readily enforced by the blockchain i.e. makes use of Munhumutapa distributed applications.
